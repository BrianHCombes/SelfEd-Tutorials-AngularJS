► Up to this point I have NOT come across much discussion about the nuances about using the url property. But its value is treated as a url fragment or a path parameter because it gets a hash mark inserted between the absolute url path and the url part as defined in this url-property:value in the code of this config module.

► In State 3, here, there is no url property defined. See Code Block CCC. I do know the presence of the url property affects browser behavior. When present it can define the default view (as in State 1). When present it's remembered as a unique url that IS recognized by the back and forward buttons. Also, it appears to be treated as a relative url within the confines of ui-router.Without the url property the view (state) will NOT be recognized by the back or forward button as is the case with this State. Also, if not present, there's no cursor pointer when hovering over it.

► Aside from these observations, maybe it's useful when evaluating the current state in order to load the correct next state.
